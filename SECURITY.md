# Security Policy

## Reporting A Vulnerability

Do not open a public issue for suspected secrets, credential exposure, command injection, supply-chain risk, or other sensitive security concerns.

Report security issues privately through GitHub's private vulnerability reporting if it is available for this repository. If private reporting is unavailable, contact the repository owner directly through the GitHub profile linked from the repository.

## Supported Scope

Security review for Mithril covers:

- packaged workflow files, skills, commands, personas, hooks, and setup docs
- GitHub Actions workflows and release automation
- dependency metadata and package contents
- host integration guidance that could affect local command execution or credentials

Mithril is currently a private npm package and should not be publicly published without an explicit release decision.
